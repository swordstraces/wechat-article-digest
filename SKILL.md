# 微信公众号文章摘要技能

## 功能概述

深度阅读微信公众号文章，按**漏斗模型**（10 秒 TLDR → 3 分钟深度价值）生成标准化摘要，并**自动入库到 IMA 知识库和笔记系统**。

## 核心特性

1. **高速抓取**：通过 `fetch_weixin.py` 直连微信，0.9s 获取完整内容（4 级回退）
2. **自动元数据**：自动提取标题、作者、公众号、字数、阅读时间
3. **结构化总结**：按固定模板生成 7 大模块的标准化摘要
4. **行动导向**：重点关注「对我的价值」，给出可执行的行动建议
5. **自动分类入库**：
   - ✅ 原文 → IMA 知识库（import_urls，按分类文件夹存放）
   - ✅ 摘要 → IMA 笔记（import_doc，分类+标签内嵌在正文中）
   - ✅ 7 大主题分类文件夹
   - ⚠️ IMA 不支持原生 tags，标签以 `#标签` 文字形式内嵌在笔记正文中
6. **知识沉淀**：摘要笔记可被 IMA 全文检索，支持后续知识关联

## 主题分类体系

### 知识库分类（7 大类）

| 分类 | emoji | 覆盖范围 |
|------|-------|---------|
| **宏观策略** | 📊 | 宏观经济、大类资产、货币财政、地缘风险 |
| **行业研究** | 🏭 | 行业分析、产业链研究、公司研究 |
| **量化投资** | 📈 | 因子研究、策略回测、择时体系、风险模型 |
| **AI与科技** | 🤖 | LLM、Agent、工具链、开源项目 |
| **投资框架** | 🧠 | 投资方法论、思维模型、决策框架 |
| **产品与创业** | 🚀 | SaaS、互联网、内容创作、自动化 |
| **日常阅读** | 📖 | 其他杂项（默认分类） |

### 标签体系（笔记正文内嵌）

> ⚠️ IMA 的知识库和笔记 API 均**不支持原生 tags 字段**。标签以 `#标签` 文字形式内嵌在笔记 Markdown 正文中，后续可通过 IMA 全文检索找到。

- **类型标签**：#宏观报告 #行业研究 #策略框架 #技术分析 #观点评论 #深度长文
- **主题标签**：由文章内容自动生成 + 用户自定义
- **重要度**：⭐ ~ ⭐⭐⭐⭐⭐
- **可执行性**：高 / 中 / 低

## 摘要结构（漏斗模型）

```
TLDR (10秒) → 核心方法论 (30秒) → 核心结论 (1分钟)
    → 核心观点 (2分钟) → 关键洞察 → 对我的价值 (3分钟)
    → 标签 + 入库信息
```

| 模块 | 回答的问题 | 时间 |
|------|-----------|------|
| ⚡ **TLDR** | 这篇文章在说什么？结论是什么？ | 10 秒 |
| 🔬 **核心方法论/假设** | 作者是怎么得出结论的？ | 30 秒 |
| 📌 **核心结论** | So What? 我该怎么做？ | 1 分钟 |
| 🎯 **核心观点** | Why? 支撑结论的论证过程 | 2 分钟 |
| 💡 **关键洞察** | 有什么反常识的发现？ | — |
| 🔥 **对我的价值** | 我能怎么用？具体做什么？ | 3 分钟 |
| 📎 **标签** | 怎么分类？ | — |
| 📂 **入库信息** | 存到哪里？什么标签？ | — |

## 触发条件

- 用户发送微信公众号链接（`mp.weixin.qq.com`）
- 用户说「总结这篇微信文章」
- 用户说「帮我读一下这篇公众号文章」

## 使用流程

### 完整流程（推荐）

```
用户发链接 → fetch-and-digest.sh 抓取文章
           → Agent 按模板生成摘要
           → Agent 确定分类和标签
           → save-to-ima.py 入库（原文 + 笔记）
           → 返回入库确认
```

### Step 1: 抓取文章

```bash
bash /root/.picoclaw/workspace/skills/wechat-article-digest/scripts/fetch-and-digest.sh \
  "https://mp.weixin.qq.com/s/xxxxx"
```

### Step 2: Agent 生成摘要

按照 `summarize-template.md` 模板和 `summary-guide.md` 指南生成摘要。

**关键**：在生成摘要时，根据文章内容确定：
1. **分类**（7 大类选 1）
2. **文章类型**（宏观报告/行业研究/策略框架/技术分析/观点评论/深度长文）
3. **重要度**（1-5 星）
4. **标签**（3-8 个具体标签）

### Step 3: 入库 IMA

将摘要保存为 Markdown 文件后执行：

```bash
python3 /root/.picoclaw/workspace/skills/wechat-article-digest/scripts/save-to-ima.py \
  --url "https://mp.weixin.qq.com/s/xxxxx" \
  --category "AI与科技" \
  --tags "知识管理,Karpathy,LLM,知识库" \
  --title "[AI与科技] Karpathy知识库方法论落地实践" \
  --digest-file /tmp/digest_xxx.md
```

#### save-to-ima.py 参数说明

| 参数 | 必需 | 说明 |
|------|------|------|
| `--url` | ✅ | 微信文章链接 |
| `--category` | 推荐 | 7 大分类之一，不传则默认「日常阅读」 |
| `--tags` | 推荐 | 额外标签，逗号分隔（自动加 # 前缀） |
| `--title` | 推荐 | 笔记标题，建议带分类前缀如 `[AI与科技] xxx` |
| `--digest-file` | 可选 | 摘要文件路径，传入则创建笔记 |
| `--dry-run` | 可选 | 只分析分类不执行入库 |

#### 仅导入原文（不创建笔记）

```bash
python3 /root/.picoclaw/workspace/skills/wechat-article-digest/scripts/save-to-ima.py \
  --url "https://mp.weixin.qq.com/s/xxxxx" \
  --category "宏观策略"
```

## 注意事项

1. **TLDR 必须有**：即使文章很长，TLDR 也要控制在 80 字以内
2. **方法论不能跳过**：说明作者是怎么得出结论的，这对判断结论可靠性至关重要
3. **「对我的价值」要用 checklist**：每个价值点用 `- [ ]` 格式，方便后续追踪
4. **标签要具体**：避免用「文章」「阅读」这类宽泛标签
5. **洞察宁缺毋滥**：没有真正独特的洞察就省略，不要凑数
6. **分类由 Agent 判断**：根据文章内容选择最匹配的 7 大分类
7. **重复检测**：同一 URL 多次入库会自动跳过原文导入
